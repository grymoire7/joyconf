# User System Design

**Date:** 2026-04-12

## Overview

This document defines the user authentication, authorization, and plan enforcement
system for Speechwave. The goal is to support multiple speakers on the free tier
while laying a clean foundation for pro, org, and billing phases without
pre-building what isn't needed yet.

---

## Scope

**In scope:**

- User registration, login, password reset, email confirmation (`mix phx.gen.auth`)
- `plan` and `is_admin` fields on the user record
- Talks owned by users (add `user_id` FK to `talks`)
- Speaker dashboard at `/dashboard` replacing the current Basic Auth `/admin`
- Plan limit enforcement wired to the existing `Plans` module
- "Coming soon" UI for pro and org tiers

**Explicitly deferred:**

- Organization model and `organization_memberships` table
- Event model
- Organizer role and organizer-specific features
- Promo/coupon codes
- Billing and Stripe integration
- Super-admin management routes (`/admin/users`, etc.)

---

## Data Model

### Generated by `mix phx.gen.auth`

```
users
  id               bigint PK
  email            string NOT NULL UNIQUE
  hashed_password  string NOT NULL
  confirmed_at     utc_datetime
  inserted_at      utc_datetime
  updated_at       utc_datetime

users_tokens
  id          bigint PK
  user_id     bigint FK → users
  token       binary NOT NULL
  context     string NOT NULL
  sent_to     string
  inserted_at utc_datetime
```

### Additional migration

```elixir
# Add plan and admin flag to users
alter table(:users) do
  add :plan, :string, null: false, default: "free"
  add :is_admin, :boolean, null: false, default: false
end

# Associate talks with their owner
# Existing rows cleared before constraint is added (pre-launch, no real data)
execute("DELETE FROM talks")
alter table(:talks) do
  add :user_id, references(:users, on_delete: :delete_all), null: false
end
```

### Design decisions

- **No `role` field on users.** "Speaker" is what every registered user is by default —
  no field needed. "Organizer" is an org-membership concept and belongs on
  `organization_memberships.role` when that table is created. Adding a global role now
  would require rework when org-scoped roles arrive.
- **`plan` stored as string**, cast to atom in the schema. Plays directly with
  `Plans.check/3` which already accepts `:free | :pro | :org`.
- **`on_delete: :delete_all`** on the talks FK — a speaker's talks are their personal
  content and should be removed if they delete their account.

## Accounts Context

`mix phx.gen.auth` generates the full `Accounts` context. We extend it with:

### `Scope` struct

```elixir
defmodule Speechwave.Accounts.Scope do
  defstruct [:user]
end
```

Holds the current authentication context. Threaded through the app as
`@current_scope`. Designed to be extended with `:org_membership` when orgs arrive
without changing any function signatures.

### `User` schema additions

```elixir
field :plan, Ecto.Enum, values: [:free, :pro, :org], default: :free
field :is_admin, :boolean, default: false
```

### Plan changeset

A dedicated `plan_changeset/2` separate from the registration changeset, so plan
changes are an explicit operation rather than something that could slip through a
general user update:

```elixir
def plan_changeset(user, attrs) do
  user
  |> cast(attrs, [:plan])
  |> validate_required([:plan])
  |> validate_inclusion(:plan, [:free, :pro, :org])
end
```

### `set_user_plan/2`

```elixir
def set_user_plan(%User{} = user, plan) when plan in [:free, :pro, :org] do
  user |> User.plan_changeset(%{plan: plan}) |> Repo.update()
end
```

Called by Stripe webhooks when billing lands. For now, used only in seeds.

## Talks Context

All public functions gain a `scope` first argument and filter to the current user.
Ownership is enforced in the query — not in the LiveView.

```elixir
def list_talks(%Scope{user: user}) do
  Repo.all(from t in Talk, where: t.user_id == ^user.id, order_by: [desc: t.inserted_at])
end

def get_talk!(%Scope{user: user}, id) do
  Repo.get_by!(Talk, id: id, user_id: user.id)
end

def create_talk(%Scope{user: user}, attrs) do
  %Talk{user_id: user.id}
  |> Talk.changeset(attrs)
  |> Repo.insert()
end
```

`get_talk!` with a mismatched `user_id` raises `Ecto.NoResultsError`, which Phoenix
renders as a 404. No separate authorization check needed.

All other `Talks` functions (`delete_talk`, `start_session`, `stop_session`,
`list_sessions`, `rename_session`, `delete_session`) receive the same scope
first-argument treatment.

## Router Structure

```elixir
# PUBLIC — no auth required
scope "/", SpeechwaveWeb do
  pipe_through :browser
  get "/", PageController, :home
  live "/t/:slug", TalkLive   # audience reaction view
end

# GENERATED — registration and login (redirect if already authenticated)
scope "/", SpeechwaveWeb do
  pipe_through [:browser, :redirect_if_user_is_authenticated]
  live_session :redirect_if_user_is_authenticated, ... do
    live "/users/register", UserRegistrationLive, :new
    live "/users/log_in", UserLoginLive, :new
    live "/users/reset_password", UserForgotPasswordLive, :new
    live "/users/reset_password/:token", UserResetPasswordLive, :edit
  end
end

# AUTHENTICATED — requires login
scope "/", SpeechwaveWeb do
  pipe_through :browser
  live_session :require_authenticated_user,
      on_mount: [{SpeechwaveWeb.UserAuth, :ensure_authenticated}] do
    live "/dashboard", DashboardLive
    live "/users/settings", UserSettingsLive, :edit
    live "/users/settings/confirm_email/:token", UserSettingsLive, :confirm_email
  end
end
```

- `TalkLive` at `/t/:slug` is fully public — audience members never log in.
- Any LiveView inside `:require_authenticated_user` receives `@current_scope`
  automatically and redirects to `/users/log_in` if unauthenticated.
- Generated auth routes are kept at their generated paths.

## Dashboard LiveView

`DashboardLive` replaces `AdminLive`. It carries over all talk and session
management functionality from `AdminLive`, scoped to the current user's talks via
`Talks.list_talks(current_scope)`.

### Super-admin UI

Admin-only items render conditionally within the same dashboard:

```heex
<%= if @current_scope.user.is_admin do %>
  <%!-- future: links to /admin/users etc. --%>
<% end %>
```

The admin-only routes are deferred. The `is_admin` field is in place for when
they're needed.

### Plan enforcement

**Session start:** before `Talks.start_session/2` is called, the LiveView checks:

```elixir
Plans.check(:full_sessions_per_month, current_scope.user.plan, session_count)
```

On `{:error, :limit_reached}`, an inline message is shown — no silent failure.

**Participant join:** `ReactionChannel` checks on socket join:

```elixir
Plans.check(:max_participants, current_scope.user.plan, current_count)
```

On limit reached, the join is rejected with a clear error sent to the client.

### "Coming soon" plan upgrade UI

A non-blocking callout in the dashboard surfaces the pro tier without gating
anything. No interaction required — just sets the expectation for future billing.

---

## Email Confirmation

`mix phx.gen.auth` generates email confirmation support. Default behavior is kept:
users can log in immediately after registering; a banner prompts email confirmation.
No features are gated behind `confirmed_at` for now.

## Seed Data

`priv/repo/seeds.exs` creates an initial admin user on fresh deploys:

```elixir
{:ok, user} = Accounts.register_user(%{
  email: "tracy@speechwave.live",
  password: System.fetch_env!("ADMIN_SEED_PASSWORD")
})

Speechwave.Repo.update!(Ecto.Changeset.change(user, is_admin: true))
```

Password sourced from an environment variable — not hardcoded.

## Removed

- `SpeechwaveWeb.AdminAuth` plug — deleted
- `:admin` pipeline in the router — deleted
- `AdminLive` — deleted, replaced by `DashboardLive`
- All `/admin` routes — removed

## Summary

| Concern | Decision |
|---|---|
| Auth generator | `mix phx.gen.auth Accounts User users` |
| User fields added | `plan` (string/enum, default `"free"`), `is_admin` (boolean) |
| Role field | None — speaker is implicit; organizer is deferred to org memberships |
| Talks ownership | `user_id` FK added; existing test data cleared in migration |
| Old admin | Removed; `/dashboard` replaces it |
| Org model | Fully deferred |
| Plan enforcement | `Plans.check/3` at session start and participant join |
| Pro/Org UI | "Coming soon" callout only |
| Email confirmation | Enabled, not required to access features |
